# Justin's Voice - Writing Guidelines

## Overview

When writing emails, messages, posts, or updates **in Justin's voice**, follow these proven patterns and rules. This guide captures Justin Strackany's distinctive writing style so you can generate authentic-sounding content.

## Core Voice Principles

### 1. Vulnerability First
**Start by sharing what you got wrong, what you're struggling with, or what surprised you.**

‚ùå Don't say: "We're excited to announce our new feature"
‚úÖ Do say: "I thought this feature would take 6 months. Built it in 3 weeks. Here's what I learned..."

**Examples:**
- "I used to think X. I was wrong."
- "Three weeks ago I had no idea how to..."
- "Everyone told me this wouldn't work."
- "I'm struggling with..."
- "Here's what I got wrong about..."

### 2. Radical Specificity
**Use exact numbers, dates, names, and details. Never say "soon" or "many" or "recently".**

‚ùå Don't say: "We've had great growth recently"
‚úÖ Do say: "We went from 12 users on Nov 1st to 847 users today (Nov 22nd)"

‚ùå Don't say: "Several people asked about this"
‚úÖ Do say: "Scott, Becky, and 3 other investors asked the same question"

**Be specific about:**
- Dates: "Dec 4th" not "next week"
- Numbers: "$847" not "under $1000"
- Names: "Becky" not "an investor"
- Time: "18 minutes" not "less than half an hour"

### 3. Conversational Tone
**Write exactly like you talk. Use contractions. Use fragments. Sound human.**

‚ùå Don't say: "We have identified several optimization opportunities"
‚úÖ Do say: "Found 3 ways to make this faster"

‚ùå Don't say: "I would appreciate your feedback"
‚úÖ Do say: "What do you think?"

**Conversational markers:**
- "Here's the thing..."
- "Turns out..."
- "That's on me"
- "Real talk:"
- "Quick question:"

### 4. Show, Don't Tell
**Tell stories and give examples instead of making claims.**

‚ùå Don't say: "Our platform is powerful and easy to use"
‚úÖ Do say: "Yesterday a non-technical founder set this up in 8 minutes. No help needed."

‚ùå Don't say: "I'm passionate about this problem"
‚úÖ Do say: "I've spent 47 hours this week on this. Missed 2 dinners. Can't stop thinking about it."

## Formatting Rules

### Line Breaks
**Single line break between paragraphs. Never more than 3 lines in a paragraph.**

‚úÖ Good:
```
I thought this would be easy.

Turns out, it's not.

Here's what I learned...
```

‚ùå Bad:
```
I thought this would be easy. Turns out, it's not. Here's what I learned about the challenges...
```

### Bold Sparingly
**Only bold 1-2 key phrases per message. Don't bold entire sentences.**

‚úÖ Good: "We're launching **Dec 4th** with Scott."
‚ùå Bad: "**We're launching Dec 4th with Scott.**"

### Bullets Over Paragraphs
**When listing things, always use bullets or numbers.**

‚úÖ Good:
```
Here's what's working:
- Founder OS shipped this weekend
- 7 contexts seeded
- API tested and working
```

‚ùå Bad:
```
Here's what's working: Founder OS shipped this weekend, we have 7 contexts seeded, and the API is tested and working.
```

### No Jargon
**Replace business-speak with normal words.**

- ‚ùå "leverage" ‚Üí ‚úÖ "use"
- ‚ùå "utilize" ‚Üí ‚úÖ "use"
- ‚ùå "bandwidth" ‚Üí ‚úÖ "time"
- ‚ùå "circle back" ‚Üí ‚úÖ "follow up"
- ‚ùå "synergy" ‚Üí ‚úÖ delete this word
- ‚ùå "at scale" ‚Üí ‚úÖ "for lots of people"

## Template Patterns

### Opening Patterns

**Choose ONE to start your message:**

**O1: The Vulnerable Opening**
```
I used to think [belief].
I was wrong.
```

**O2: The Context-Shift Opening**
```
Everyone's doing [common thing].
We're doing [different thing].
Here's why...
```

**O3: The Contrarian Opening**
```
[Popular opinion].

I disagree.
```

**O4: The Direct Opening**
```
Quick question:
[Your question]
```

**O5: The Time-Stamp Opening**
```
[X days/weeks/months] ago, [situation].
Today, [different situation].
```

**O6: The Admission Opening**
```
I haven't [done something] in [time period].
That's on me.
```

### Middle Patterns

**Choose ONE to develop your point:**

**M1: Story Arc**
```
Here's what happened:
[Chronological story with specific details]
```

**M2: Pattern Recognition**
```
I've noticed [pattern].

Every time [X happens], [Y happens].

Example: [Specific story]
```

**M3: Framework**
```
Here's how I think about this:

1. [First principle]
2. [Second principle]
3. [Third principle]

[Apply to current situation]
```

**M4: The Obstacle Course**
```
We tried [approach 1]. Didn't work because [specific reason].
Then tried [approach 2]. Failed because [specific reason].
Finally figured out [approach 3].
```

**M5: The Contrast**
```
Most people think [X].
But actually [Y].

Here's the difference:
[Specific comparison]
```

### Ending Patterns

**Choose ONE to close:**

**E1: The Invitation**
```
Want to [do something specific]?

[Clear next step]
```

**E2: The Cliffhanger**
```
More on this [timeframe].

For now: [one thing to remember]
```

**E3: The Mic Drop**
```
[Provocative one-liner that summarizes everything]
```

**E4: The CTA**
```
If you want [outcome]:
Do [specific action]

[Link or next step]
```

**E5: The Reflection**
```
Lessons learned:
- [Lesson 1]
- [Lesson 2]
- [Lesson 3]
```

**E6: The Simple Ask**
```
Quick question:
[Specific question]
```

## Proven Blend Recipes

### For Investor Updates
**Opening:** O1 (Vulnerable)
**Middle:** M1 (Story Arc)
**Ending:** E2 (Cliffhanger)

Example:
```
I thought we'd have this demo done a week ago.
We didn't.

Here's what happened:
[Story of what took longer and why]

Demo is now scheduled for Dec 4th with Scott.

More after that meeting.
```

### For Product Launches
**Opening:** O2 (Context-Shift)
**Middle:** M5 (Contrast)
**Ending:** E4 (CTA)

Example:
```
Everyone's building AI to automate your work.

We built one to have better conversations with you.

Most AI tools want to replace you.
Ours wants to understand you.

If you want to try it:
[Link]
```

### For Personal Outreach
**Opening:** O6 (Admission)
**Middle:** M2 (Pattern)
**Ending:** E1 (Invitation)

Example:
```
Haven't talked in 3 weeks.
That's on me.

I've noticed when I get heads-down on a demo,
I lose track of everything else.

This week: happened again.

Want to catch up Friday?
```

### For Team Communication
**Opening:** O4 (Direct)
**Middle:** M3 (Framework)
**Ending:** E5 (Reflection)

Example:
```
Quick question:
How should we prioritize this sprint?

Here's how I think about it:
1. User pain > New features
2. Bugs that block > Bugs that annoy
3. What we promised > What we want

Based on that: fix the auth bug first.

Lessons from last sprint:
- We over-committed
- We under-estimated
- We shipped anyway

Let's not do that again.
```

## Voice Checklist

Before sending, check for:

- [ ] **Vulnerability**: Did I share what I got wrong or what surprised me?
- [ ] **Specificity**: Did I use exact dates, numbers, and names?
- [ ] **Conversational**: Does this sound like how I actually talk?
- [ ] **Short paragraphs**: Are all paragraphs 3 lines or less?
- [ ] **Show don't tell**: Did I use stories instead of claims?
- [ ] **No jargon**: Did I replace business-speak with normal words?
- [ ] **Clear next step**: Is it obvious what happens next?
